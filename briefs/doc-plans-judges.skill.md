# # Skill: Doc-Plans-Judges 

# ## Purpose 

Create, refine, and organize project documents: 

- master product plans, 

- judges systems, 

- research summaries, 

- competitive analyses. 

# ## When to use 

- Turning messy notes into a structured plan. 

- Creating new docs for the project. 

- Updating existing docs (adding sections, cleaning structure). 

# ## Inputs 

- Goal of the document (one sentence). 

- Raw notes or existing text. 

- Optional: audience (you, investors, collaborators, AI). 

# ## Core principles 

- Simple, scannable structure. 

- Clear headings and short paragraphs. 

- Visible "How to use this later" section. 

- No fluff; every section supports the project. 

# ## Behavior 

When this skill is active: 

1. **Clarify the document’s job.** 

- Restate the goal in one sentence. 

- Identify which sections are useful and which are noise. 

2. **Organize content.** 

- Create a section structure (H2/H3). 

- Group related ideas. 

- Remove duplicates and tangents. 

3. **Rewrite into Markdown.** 

- Use GitHub-Flavored Markdown. 

- Headings, bullets, short paragraphs. 

- Optional: small tables for comparisons. 

4. **Add usage instructions.** 

- At the end, add an "How to use this document" section. 

- Explain how future-you should use it. 

# ## Output format 

- Title as H1. 

- 3–6 main sections (H2). 

- Subsections when needed (H3). 

- Short paragraphs and lists. 

- Final section: "How to Use This Document". 

# ## Example prompt 

Mode: Doc-Plans-Judges Task: Turn my notes into a master plan doc. 

Goal: 

> Master plan for FocusReader ADHD reading-to-audio product. 

Raw notes: 

> [paste messy notes here] 

Instructions: 

- Clean it up. 

- Use sections. 

- End with "How to use this document". 

# ## Success criteria 

- The document can be read in under 10 minutes. 

- Future-you understands why it exists and how to use it. 

- It’s easy to copy into tools (Claude, VS Code, Obsidian).
