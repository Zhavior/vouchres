# SubscriberHub.tsx patch

## Problem

`src/components/SubscriberHub.tsx` lines 93-127 contain a hardcoded
`cappers` array with fabricated win rates (72.8%, 64.1%) and subscriber
counts (1,840, 752) for "Alpha Baseball Guru" and "Homers & Parabolics".
These cappers don't exist in the system — they're fictional.

## Fix

Replace the hardcoded array with a fetch to `/api/cappers`, which queries
the real `public.cappers` table seeded in `schema.sql`. All demo cappers
get a "DEMO" badge so users know they're not real people.

### Backend endpoint to add

```ts
coreRoutes.get("/cappers", async (req, res) => {
  const { supabaseAdmin } = await import("../middleware/auth");
  const { data, error } = await supabaseAdmin
    .from("cappers")
    .select(`
      id,
      display_name,
      tagline,
      is_demo,
      is_active,
      trust_scores!inner(score, total_picks, won_picks, lost_picks, net_units)
    `)
    .eq("is_active", true)
    .eq("trust_scores.scope", "overall")
    .order("trust_scores.score", { ascending: false });

  if (error) return res.status(500).json({ error: "fetch_failed" });

  return res.json({
    cappers: (data ?? []).map((c: any) => ({
      id: c.id,
      display_name: c.display_name,
      tagline: c.tagline,
      is_demo: c.is_demo,
      trust_score: c.trust_scores[0]?.score ?? 50,
      total_picks: c.trust_scores[0]?.total_picks ?? 0,
      won_picks: c.trust_scores[0]?.won_picks ?? 0,
      lost_picks: c.trust_scores[0]?.lost_picks ?? 0,
      net_units: c.trust_scores[0]?.net_units ?? 0,
      // subscriber_count: 0 — implement when paid subs exist
    })),
  });
});
```

### UI changes for demo cappers

Render every capper with `is_demo: true` with:

- A `DEMO` badge next to their name (visible at all times, not on hover)
- A tooltip: "This is an automated demo capper. Picks are generated by the
  server-side agent, not a real person."
- No "Subscribe for $X/mo" button — demo cappers cannot be subscribed to
  until paid subscriptions are actually implemented and legally reviewed

### Also remove

- The "Subscribe to unlock premium picks" CTA — there is no subscription
  flow yet. Replace with "Follow" (free) and a disabled "Subscribe" button
  with tooltip "Coming soon".
- Any "Subscribers: 1,840" labels — replace with "Followers: 0" (real data).
